## 标准发布流程（3 步）

### 第 1 步：发布到 NuGet

```bash
cd /Users/dingyuwang/0-X/iot-sdk

# 先 dry-run 验证（必须！）
zl-pipeline publish 1.0.X --dry-run

# 全部通过后正式发布（build → pack → obfuscar → API对比 → 推送NuGet）
zl-pipeline publish 1.0.X
```

**pipeline.json 配置位置**：`iot-sdk/pipeline.json`
- 22 个项目已全部配置
- consumers 已配置 tmom 下游

### 第 2 步：同步下游消费项目

```bash
cd /Users/dingyuwang/0-X/iot-sdk

# dry-run 预览变更（必须！）
zl-pipeline sync-consumers 1.0.X --dry-run

# 确认无误后执行同步
zl-pipeline sync-consumers 1.0.X
```

**sync-consumers 自动完成**：
1. 扫描 artifacts/ 中对应版本的 nupkg
2. 更新 tmom `Directory.Packages.props` 中的版本号
3. 编译验证 tmom 可正常构建
4. （可选）自动 git commit + push（需设置 `"autoCommit": true`）

### 第 3 步：验证

```bash
# 验证 NuGet 上发布的包
zl-pipeline check ZL.Dao.IotDevice 1.0.X

# 验证 iot-sdk 自身编译
zl-pipeline verify 1.0.X
```

## pipeline.json 配置说明

```jsonc
{
  "$schema": "https://raw.githubusercontent.com/qwdingyu/ZL.Pipeline/main/schemas/pipeline-schema.json",
  "version": "1.0",
  "projects": [
    {
      "name": "ZL.Dao.IotDevice",          // 项目名
      "csproj": "src/domain/ZL.Dao.IotDevice/ZL.Dao.IotDevice.csproj",  // csproj 相对路径
      "obfuscate": true                     // 是否混淆
    }
    // ... 22 个项目
  ],
  "nugetSource": "https://api.nuget.org/v3/index.json",
  "publishTimeout": 120,
  "dryRun": false,
  "consumers": [                            // ← 下游消费项目
    {
      "name": "tmom",                      // 消费项目名称
      "path": "/Users/dingyuwang/0-X/tmom", // 项目根目录（绝对路径）
      "cpmFile": "Directory.Packages.props", // CPM 文件路径
      "buildTarget": "api/TMom.Device.Runtime.Host/TMom.Device.Runtime.Host.csproj", // 编译验证目标
      "autoCommit": false                  // true=自动 git commit+push
    }
  ]
}
```

## 常见问题

### Q: sync-consumers 报告 "未找到 nupkg"
A: 先运行 `zl-pipeline publish` 生成 artifacts/ 目录下的 nupkg。

### Q: sync-consumers 编译验证失败
A: 说明版本升级引入了破坏性变更。先修复代码，再重新发布。

### Q: autoCommit 能否设为 true？
A: 前提是下游项目 git remote 可正常 push（无权限问题）。tmom 在 Gitee 上需确保有推送权限。

### Q: 新消费项目如何接入？
A: 在 `pipeline.json` 的 `consumers` 数组中添加配置项即可。

## 历史教训（不再犯的错）

| 错误 | 对策 |
|---|---|
| 手动 `dotnet build/pack/push` | 必须用 `zl-pipeline publish` |
| 手动 `cp` nupkg 到 local-feed | 从 nuget.org 直接拉取，不需要本地 feed |
| 手动编辑 csproj 切换引用方式 | csproj 的切换是一次性变更，完成后不再改动 |
| 手动编辑 Directory.Packages.props | 必须用 `zl-pipeline sync-consumers` |
| 忘记 git push | 后续通过 `autoCommit` 或 CI 自动化解决 |
| 忘记先 dry-run | 所有操作必须先 `--dry-run` 再执行 |
