# ZL.Watchdog 设计理念与架构文档

**文档编号**：03
**日期**：2026-06-05
**项目**：`src/platform/ZL.Watchdog`
**目标框架**：netstandard2.1

---

## 1. 设计动机

### 1.1 问题背景

在工业 IoT 和 PLC 仿真场景中，协议服务器（S7、Modbus、OPC UA 等）需要 7×24 小时运行。
服务器崩溃后如果没有自动恢复机制，将导致：

- 数据采集链路中断，上位系统无感知
- 运维人员需要人工巡检发现故障
- 生产环境服务中断时间不可控

### 1.2 为什么独立为 SDK

最初 watchdog 逻辑嵌入在 PlcSimulator.Core 的 `BatchOrchestrator` 中，
与 `IDeviceServer`、`ProtocolKind`、`ServerFactory` 等 PLC 仿真器概念紧密耦合。

这导致：

| 问题 | 影响 |
|------|------|
| 不可复用 | IoT 网关、微服务集群等项目无法使用 |
| 增加 Core 复杂度 | 协议引擎库不应承载运维关注点 |
| 测试耦合 | watchdog 测试依赖 PLC 仿真器类型 |

**决策**：将 watchdog 抽取为独立的、通用的 `ZL.Watchdog` SDK，
放置在 `iot-sdk/src/platform/` 下，作为平台级基础设施。

## 2. 核心设计理念

### 2.1 零业务耦合

ZL.Watchdog 不知道也不关心它监控的是什么：

- ❌ 不知道 PLC、协议、端口、内存模型
- ❌ 不依赖任何工业通信库
- ❌ 不引用任何业务框架

它只知道三件事：
1. **组件有名称**（`string Name`）
2. **组件可以被检查是否健康**（`HealthCheckFunc → bool`）
3. **组件可以被重启**（`RestartAction → bool`）

### 2.2 委托驱动

调用方通过两个委托注入业务逻辑：

```
HealthCheckFunc  ──→  调用方决定"什么是健康"
RestartAction    ──→  调用方决定"如何重启"
```

这意味着同一个 `WatchdogService` 可以用于：

| 场景 | HealthCheck | Restart |
|------|-------------|---------|
| PLC 仿真器 | `server.IsRunning` | `Stop() → new Server() → Start()` |
| IoT 网关 | `tcpClient.Connected` | `Reconnect()` |
| Web 服务 | `HttpClient.GetAsync("/health")` | `Process.Start()` |
| 数据库连接池 | `connection.Ping()` | `connection.Open()` |

### 2.3 滑动窗口限流

防止"重启风暴"——当一个组件反复崩溃时，无限重启会耗尽系统资源。

```
窗口 60 秒内最多重启 3 次

时间线:
  0s   崩溃 → 重启 #1
  12s  崩溃 → 重启 #2
  25s  崩溃 → 重启 #3
  38s  崩溃 → 已达上限，放弃重启，触发告警
  61s  窗口滑动，重启 #1 过期 → 恢复重启能力
```

### 2.4 事件通知

两个事件让调用方接入自己的日志/告警系统：

```csharp
watchdog.HealthCheckFailed += (_, e) =>
{
    // e.Name           组件名称
    // e.RestartTriggered  是否触发了重启
    // e.SkipReason     未重启的原因
};

watchdog.Restarted += (_, e) =>
{
    // e.Name           组件名称
    // e.Success        重启是否成功
    // e.RestartCount   窗口内累计重启次数
};
```

## 3. 架构

### 3.1 类型关系

```
WatchdogService (主服务)
  │
  ├── _entries: ConcurrentDictionary<string, EntryState>
  │     │
  │     └── EntryState
  │           ├── WatchedEntry (公开类型)
  │           │     ├── Name: string
  │           │     ├── HealthCheck: HealthCheckFunc
  │           │     ├── Restart: RestartAction
  │           │     └── AutoRestart: bool
  │           ├── RestartTimestamps: List<DateTime>
  │           └── Lock: object
  │
  ├── _options: WatchdogOptions
  │     ├── CheckIntervalSeconds: int
  │     ├── WindowSeconds: int
  │     └── MaxRestartsPerWindow: int
  │
  ├── Events
  │     ├── HealthCheckFailed: EventHandler<HealthCheckFailedEventArgs>
  │     └── Restarted: EventHandler<RestartedEventArgs>
  │
  └── Metrics
        ├── TotalChecks: long
        ├── TotalRestarts: long
        └── TotalAlerts: long
```

### 3.2 执行流程

```
Start()
  │
  └── Task.Run(LoopAsync)
        │
        └── while (!cancelled)
              │
              ├── Task.Delay(CheckIntervalSeconds)
              │
              └── CheckAll()
                    │
                    └── foreach entry in _entries
                          │
                          ├── entry.HealthCheck()
                          │     ├── true  → 跳过
                          │     └── false → HandleUnhealthy()
                          │
                          └── HandleUnhealthy()
                                │
                                ├── PruneTimestamps()  (清理过期重启记录)
                                │
                                ├── AutoRestart=false? → 告警, 返回
                                │
                                ├── 已达限流上限?     → 告警, 返回
                                │
                                └── 执行重启
                                      ├── entry.Restart(name)
                                      ├── 记录时间戳
                                      └── 触发 Restarted 事件
```

### 3.3 线程安全模型

| 组件 | 保护机制 | 说明 |
|------|---------|------|
| `_entries` | `ConcurrentDictionary` | 支持并发 Register/Unregister |
| `EntryState.RestartTimestamps` | `lock(EntryState.Lock)` | 单组件级别锁，避免全局锁竞争 |
| `TotalChecks/Restarts/Alerts` | `Interlocked` | 无锁原子计数 |

## 4. 使用示例

### 4.1 PLC 仿真器场景

```csharp
var watchdog = new WatchdogService(new WatchdogOptions
{
    CheckIntervalSeconds = 5,
    WindowSeconds = 60,
    MaxRestartsPerWindow = 3
});

foreach (var server in plcServers)
{
    var captured = server;
    watchdog.Register(new WatchedEntry(
        name: server.Name,
        healthCheck: () => captured.IsRunning,
        restart: _ =>
        {
            captured.Stop();
            var newServer = factory.Create(captured.Config);
            return newServer.Start();
        }
    ));
}

watchdog.Start();
```

### 4.2 IoT 网关场景

```csharp
var watchdog = new WatchdogService(new WatchdogOptions
{
    CheckIntervalSeconds = 10,
    WindowSeconds = 300,
    MaxRestartsPerWindow = 5
});

foreach (var connection in gatewayConnections)
{
    var conn = connection;
    watchdog.Register(new WatchedEntry(
        name: conn.DeviceId,
        healthCheck: () => conn.IsConnected,
        restart: _ =>
        {
            conn.Disconnect();
            conn.Connect();
            return conn.IsConnected;
        }
    ));
}

watchdog.HealthCheckFailed += (_, e) =>
    logger.Warning("Gateway connection {Name} failed: {Reason}",
        e.Name, e.SkipReason ?? "restarting");

watchdog.Start();
```

### 4.3 仅监控不重启

```csharp
// 不提供 restart 委托 → AutoRestart = false
watchdog.Register(new WatchedEntry(
    name: "database",
    healthCheck: () => dbConnection.Ping()
));

watchdog.HealthCheckFailed += (_, e) =>
    alertService.Send($"Database unhealthy: {e.Name}");
```

## 5. 配置参考

### 5.1 WatchdogOptions

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `CheckIntervalSeconds` | int | 5 | 健康检查间隔（秒），最小 1 |
| `WindowSeconds` | int | 60 | 滑动窗口时长（秒），必须 ≥ CheckIntervalSeconds |
| `MaxRestartsPerWindow` | int | 3 | 窗口内最大重启次数，最小 1 |

### 5.2 配置验证

```csharp
var options = new WatchdogOptions { CheckIntervalSeconds = 0 };
var errors = options.Validate();
// errors: ["CheckIntervalSeconds 必须 >= 1"]
```

构造函数会自动调用 `Validate()`，无效配置抛出 `ArgumentException`。

## 6. 状态查询

### 6.1 整体状态

```csharp
WatchdogStatus status = watchdog.GetStatus();

status.Running          // bool: 是否正在运行
status.TotalChecks      // long: 累计检查次数
status.TotalRestarts    // long: 累计重启次数
status.TotalAlerts      // long: 累计告警次数
status.Entries          // EntryStatus[]: 各组件状态
```

### 6.2 单组件状态

```csharp
EntryStatus entry = status.Entries[0];

entry.Name                // string: 组件名称
entry.Healthy             // bool: 当前是否健康
entry.RestartCount        // int: 窗口内重启次数
entry.RestartLimitReached // bool: 是否已达重启上限
entry.LastRestartAt       // DateTime?: 上次重启时间
```

## 7. 与旧实现的对比

| 维度 | 旧实现 (ServerWatchdogService) | ZL.Watchdog |
|------|------|------|
| 位置 | PlcSimulator.Core/Batch/ | iot-sdk/src/platform/ |
| 目标框架 | net8.0 | netstandard2.1 |
| 外部依赖 | IDeviceServer, ProtocolKind, PlcMemory | 无 |
| 注册方式 | Register(name, protocol, port, server, memory, options) | Register(WatchedEntry) |
| 健康检查 | 硬编码 `server.IsRunning` | 委托注入 |
| 重启逻辑 | 内置 Stop→Factory→Start | 委托注入 |
| 事件 | 无 | HealthCheckFailed + Restarted |
| 指标 | TotalChecks + TotalRestarts | + TotalAlerts |
| 可复用性 | ❌ 仅 PlcSimulator | ✅ 任何 .NET 项目 |
