## 一、Foundation 层 (6个项目)

### 1. ZL.Framing - 协议帧组装/解析库
- **包名**：`ZL.Framing`
- **文件数**：10 个 `.cs` 文件
- **目标框架**：`.NET 8.0 / .NET Standard 2.1 / .NET Standard 2.0`（多目标）
- **导出类型**：
  - `ByteFramingOptions` — 帧组装选项配置，支持 Timeout/FixedLength/LengthField 三种模式
  - `FrameAssembler` — 帧组装器主类，将字节流输入写入 StickyWindowBuffer 后解码为完整帧
  - `IByteBuffer` — 环形缓冲区接口
  - `StickyWindowBuffer` — 环形字节缓冲区实现，内部维护 byte[] 池和读写指针
  - `IFrameDecoder` — 帧解码器接口
  - `FixedLengthFrameDecoder` — 固定长度帧解码器
  - `LengthFieldFrameDecoder` — 长度字段帧解码器，支持 14 种 CRC 算法和多种长度字段类型
  - `ChecksumUtil` — 14 种 CRC 算法实现（CRC8/CRC16/CCITT/CRC32/Modbus 等）
  - `FramingHex` — 帧级别十六进制编解码工具
- **核心用途**：处理二进制协议通信中的粘包/拆包问题，将原始字节流组装成完整的数据帧
- **依赖**：无外部 ProjectReference，纯底层库
- **代码量**：约 2,500 行

### 2. ZL.DataProcessing - 数据处理与表达式引擎
- **包名**：`ZL.DataProcessing`
- **文件数**：6 个 `.cs` 文件
- **导出类型**：

  - `HexCodec` — 十六进制编解码工具
  - `DataExpressionEngine` — 基于 DynamicExpresso 的动态表达式引擎，支持运行时编译 C# 表达式
  - `TemplateParser` — Scriban 模板解析器
  - `ScriptHelpers` — 脚本工具函数
- **核心用途**：提供二进制数据处理能力，包括模式匹配、十六进制编解码、C# 动态表达式引擎和 Scriban 模板引擎
- **依赖**：`ZL.Script`、`ZL.Scripting`
- **代码量**：约 800 行

## 关键发现

1. **ZL.Iot.Runner.Templates 是纯资源库**：0 个 `.cs` 文件，所有模板以 EmbeddedResource 嵌入程序集，这是正常的 MSBuild 嵌入式资源模式。

2. **ZL.Framing 是多目标库**：同时支持 .NET 8.0、.NET Standard 2.1 和 .NET Standard 2.0，兼容性最好。

3. **ZL.DataConvert 是调用频率最高的库**：无依赖，是整个物联网协议栈中 byte[] <-> 数值转换的核心，3,500 行代码中 ByteArrayLib 占 976 行。

4. **ZL.Dao.IotDevice 是最大的项目**：78 个文件、约 8,000 行代码，覆盖 IoT 设备管理的表结构几乎全部。

5. **CLI 项目代码量极小**：两个 CLI 项目各只有约 50 行代码，本质是 Program.Main 入口。
