## 混淆决策原则

基于本次审计，确立以下 **5 条混淆决策原则**，后续新增库按此决策：

| 优先级 | 原则 | 示例 |
|--------|------|------|
| **P0** | **契约库禁止混淆** | 纯接口/枚举/DTO 的库（ZL.Iot.Interface） |
| **P0** | **被继承的基类禁止混淆** | public abstract class 供外部继承（ZL.PlcBase 的 DeviceBase） |
| **P1** | **插件宿主禁止混淆** | Assembly.LoadFrom + GetType(string) 加载外部 DLL（ZL.Iot.Plugin） |
| **P1** | **代码生成器禁止混淆** | Scriban/Template + Assembly.LoadFrom 反射（ZL.Iot.Runner.Generator） |
| **P2** | **纯内部工具库可以混淆** | 无 public 类型或被外部继承（ZL.Framing, ZL.DataProcessing） |

### 安全混淆的判断标准

| 安全（可混淆） | 不安全（禁止混淆） |
|---------------|-------------------|
| `value.GetType().Name` 字符串匹配 | `ass.GetType("命名空间.类名")` 字符串查找 |
| `typeof(T)` 编译期类型 | `Assembly.LoadFrom(path)` 运行时加载 |
| `new XmlSerializer(typeof(Config))` 自身类型 | `GetManifestResourceStream(name)` 嵌入资源 |
| `[SugarColumn]` 属性标记 | `GetGenericTypeDefinition() == typeof(OperateResult<>)` |
| 无 public 类型 | public interface/abstract class 供外部实现 |

## 混淆收益分析

### 实际有效混淆的项目（12 个）

| 项目 | 预估混淆类型数 | 保护价值 |
|------|-------------|---------|
| ZL.Shared | 中 | 日志配置细节 |
| ZL.Framing | 低 | 协议帧格式 |
| ZL.Script | 低 | 表达式解析逻辑 |
| ZL.ConnectionGuard | 中 | 看门狗策略 |
| ZL.DataProcessing | 低 | 数据处理算法 |
| ZL.DataConvert | 中 | 字节转换算法 |
| ZL.DB.Acc | 低 | ORM 封装 |
| ZL.Dao.IotDevice | 低 | 数据访问 |
| ZL.Dao.Edge | 低 | 数据访问 |
| ZL.Biz.Execute | 低 | 业务执行 |
| ZL.Iot.Runner | 中 | 运行时调度 |
| ZL.EdgeService | 中 | 边缘服务逻辑 |

### 不混淆的项目（8 个）

| 项目 | 原因 | 替代保护 |
|------|------|---------|
| ZL.Scripting | 反射在外部对象上 | 代码本身简单，无可混淆价值 |
| ZL.Iot.Interface | 契约库 | API 文档 + License 限制 |
| ZL.Iot.Plugin | 插件宿主 | 授权校验逻辑放在独立模块 |
| ZL.Iot.Runner.Generator | 代码生成器 | 工具类库，不直接分发到客户端 |
| ZL.PFLite | 可混淆 | — |
| ZL.Tag | 可混淆 | — |
| ZL.PlcBase | 基类库 | 继承链必须保持 |
| ZL.PlcBase.Bridges | 契约层 | 接口必须暴露 |

**结论**：54% 的混淆收益来自 12 个有效混淆项目。4 个不混淆的库是因为它们**必须暴露公共 API**，混淆在这些场景下不是安全问题而是正确性问题。
