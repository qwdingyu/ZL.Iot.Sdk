## 项目简介

ProtocolGateway 是一个轻量级、插件化的协议转发中间类库，专为工控领域设计，支持多种工业协议之间的数据转换和路由。

### 核心特性

- ✅ **多协议支持** - Serial/Modbus/MQTT/TCP/HTTP 等
- ✅ **插件化架构** - 输入/输出/转换器独立插件
- ✅ **灵活路由** - 基于条件的消息路由规则
- ✅ **零代码配置** - JSON/YAML 配置文件驱动
- ✅ **跨平台** - Windows/Linux/macOS
- ✅ **轻量级** - 核心库仅约 500 行代码

## 快速开始

### 安装

```bash
dotnet add package ProtocolGateway
```

### 5 分钟示例：串口数据转发到 HTTP API

```csharp
using ZL.ProtocolGateway;
using ZL.ProtocolGateway.Plugins;

var pipeline = new MessagePipeline();

// 串口输入
var serialInput = new SerialInputPlugin(new SerialInputConfig
{
    Name = "Serial-COM1",
    PortName = "COM1",
    BaudRate = 9600
});

// HTTP 输出
var httpOutput = new HttpOutputPlugin(new HttpOutputConfig
{
    Name = "HTTP-API",
    Url = "http://api.example.com/data"
});

pipeline.RegisterOutput(httpOutput);
pipeline.AddRouter(new RouteRule
{
    Condition = msg => msg.Metadata["Protocol"] == "Serial",
    OutputNames = { "HTTP-API" }
});

await pipeline.StartAsync();
await serialInput.StartAsync(msg => pipeline.ProcessAsync(msg));
```

## 开发指南

### 添加新插件

1. 实现 `IInputPlugin` 或 `IOutputPlugin` 接口
2. 在插件中处理协议特定逻辑
3. 使用 `Message` 类进行数据交换
4. 编写单元测试

### 添加新转换器

1. 实现 `IProtocolConverter` 接口
2. 在 `Convert` 方法中实现转换逻辑
3. 返回 `null` 表示转换失败（消息丢弃）
4. 编写单元测试验证转换结果

### 测试

```bash

dotnet test

# 查看测试覆盖率
dotnet test /p:CollectCoverage=true
```
