# ProtocolGateway Pipeline Transformer 使用指南

> 原 `IProtocolConverter` 接口已于 2026-05-31 移除，统一使用 `Pipeline.AddTransformer` 进行消息转换。

## 一、什么是 Pipeline Transformer

Pipeline Transformer 是 `MessagePipeline` 的消息转换机制。每条消息在路由到 OutputPlugin 之前，会依次经过所有已注册的 Transformer。

```
Message → [Transformer 1] → [Transformer 2] → … → Router → OutputPlugin
```

## 二、基本用法

### 2.1 注册单个 Transformer

```csharp
var pipeline = new MessagePipeline();

pipeline.AddTransformer(async message =>
{
    // 在这里修改 message
    return message;
});

await pipeline.StartAsync();
```

### 2.2 JSON 字段映射

```csharp
pipeline.AddTransformer(async message =>
{
    if (message.ContentType != "json")
        return message;

    using var doc = JsonDocument.Parse(message.GetJsonContent());
    var root = doc.RootElement;
    var output = new JsonObject();

    // 字段重命名
    var fieldMapping = new Dictionary<string, string>
    {
        ["device"] = "deviceId",
        ["temperature"] = "temperatureCelsius",
        ["pressure"] = "pressureMpa"
    };

    foreach (var mapping in fieldMapping)
    {
        if (root.TryGetProperty(mapping.Key, out var value))
        {
            output[mapping.Value] = JsonNode.Parse(value.GetRawText());
        }
    }

    output["timestamp"] = DateTime.Now.ToString("yyyy-MM-dd HH:mm:ss");
    message.SetJsonContent(output.ToJsonString());
    return message;
});
```

### 2.3 CSV → JSON 转换

```csharp
pipeline.AddTransformer(async message =>
{
    if (message.Metadata.TryGetValue("Protocol", out var proto) && proto == "Serial")
    {
        var rawData = message.GetTextContent();
        var parts = rawData?.Split(',');

        if (parts != null && parts.Length >= 2)
        {
            var payload = new JsonObject
            {
                ["sn"] = parts[0].Trim(),
                ["stationNo"] = "STATION-A01",
                ["timestamp"] = DateTime.Now.ToString("yyyy-MM-dd HH:mm:ss"),
                ["values"] = null!
            };

            var values = new JsonArray();
            for (int i = 1; i < parts.Length; i++)
            {
                if (double.TryParse(parts[i].Trim(), out var value))
                {
                    values.Add(new JsonObject
                    {
                        ["index"] = i,
                        ["value"] = value
                    });
                }
            }
            payload["values"] = values;

            message.SetJsonContent(payload.ToJsonString());
            message.Metadata["Protocol"] = "BridgeResult";
        }
    }

    return message;
});
```

## 三、链式组合

多个 Transformer 按注册顺序依次执行：

```csharp
// 第一级：Hex → Text
pipeline.AddTransformer(async message =>
{
    if (message.ContentType != "hex") return message;
    var text = message.GetHexContent();
    message.SetTextContent(text);
    return message;
});

// 第二级：Text → JSON
pipeline.AddTransformer(async message =>
{
    if (message.ContentType != "text") return message;
    var text = message.GetTextContent();
    var obj = new JsonObject { ["raw"] = text, ["time"] = DateTime.Now.ToString("O") };
    message.SetJsonContent(obj.ToJsonString());
    return message;
});

// 第三级：JSON 字段重命名
pipeline.AddTransformer(async message =>
{
    if (message.ContentType != "json") return message;
    using var doc = JsonDocument.Parse(message.GetJsonContent());
    var root = doc.RootElement;
    var output = new JsonObject();
    if (root.TryGetProperty("raw", out var v1))
        output["data"] = JsonNode.Parse(v1.GetRawText());
    if (root.TryGetProperty("time", out var v2))
        output["timestamp"] = JsonNode.Parse(v2.GetRawText());
    message.SetJsonContent(output.ToJsonString());
    return message;
});
```

## 四、条件转换

在 Transformer 内部根据消息属性决定是否转换：

```csharp
pipeline.AddTransformer(async message =>
{
    // 只对特定协议的消息做转换
    if (message.Metadata.TryGetValue("Protocol", out var p) && p == "Modbus")
    {
        // Modbus 特定解析逻辑
    }

    return message;
});
```

## 五、异常处理

Transformer 中应妥善处理异常，避免一条消息的转换失败影响后续消息：

```csharp
pipeline.AddTransformer(async message =>
{
    try
    {
        // 转换逻辑
    }
    catch (Exception ex)
    {
        // 记录日志，保持原消息不变
        Console.WriteLine($"转换失败: {ex.Message}");
    }

    return message;
});
```

## 六、最佳实践

1. **尽早返回** — 如果消息不需要转换，第一时间 `return message`
2. **不要改变消息顺序** — Transformer 是同步流水线，按注册顺序执行
3. **处理异常** — 转换失败时返回原消息，不要抛异常中断 Pipeline
4. **修改 Metadata** — 转换后更新 `message.Metadata["Protocol"]` 标记新协议类型
5. **使用 ContentType** — 通过 `message.ContentType` 判断消息格式，不要只看 Metadata

## 七、与旧版 IProtocolConverter 的差异

| 旧版 (已移除) | 新版 |
|---|---|
| `IProtocolConverter.Convert(Message)` | `pipeline.AddTransformer(async msg => ...)` |
| `ConverterTransformer` 适配器 | 不需要适配器，直接注册 |
| `SerialToHttpConverter` 类 | 内联 CSV→JSON Transformer |
| `JsonTransformerConverter` 类 | 内联 JSON 字段映射 Transformer |
| 需要实现接口 + 适配器 | 直接写 lambda 函数 |
