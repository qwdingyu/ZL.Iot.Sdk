## 执行摘要

本文档详细分析了 ProtocolGateway 协议转发类库的需求、痛点和解决方案。作为面向工控领域的通用协议转换中间件，目标是实现**多协议兼容、插件化扩展、零代码配置**的工业级数据网关。

### 核心目标

| 目标 | 描述 | 优先级 |
|------|------|--------|
| 多协议兼容 | 支持 Serial/Modbus/MQTT/TCP/HTTP/OPC UA 等主流工业协议 | P0 |
| 插件化架构 | 输入/输出/转换器独立插件，支持热插拔 | P0 |
| 零代码配置 | JSON/YAML 配置文件驱动，无需编译即可部署 | P1 |
| 规则引擎 | 可视化规则配置，支持复杂数据转换和路由 | P1 |
| 监控告警 | 实时运行状态监控，异常告警通知 | P2 |

### 1.2 用户痛点

**痛点 1：协议孤岛**
- 每台设备需要独立的采集程序
- 数据格式不统一，无法集中处理
- 新设备接入需要重新开发

**痛点 2：维护成本高**
- 每个项目重复开发采集逻辑
- 缺乏统一的配置管理
- 现场调试困难，问题定位耗时

**痛点 3：扩展性差**
- 硬编码协议处理逻辑
- 无法动态添加新协议
- 升级需要停机

**痛点 4：数据质量**
- 缺少数据校验和过滤
- 异常数据处理不当
- 断点续传支持不足

### 3.2 核心功能需求

#### FR-1: 多协议输入

**需求描述：** 支持从多种协议源接收数据

**验收标准：**
- 可配置多个输入源，每个源独立运行
- 支持热插拔，添加/删除输入源不影响其他源
- 自动重连机制，断线后自动恢复
- 输入源状态可监控（连接状态、接收字节数、错误计数）

**配置示例：**
```yaml
inputs:
  - name: Serial-COM1
    type: Serial
    enabled: true
    config:
      port: COM1
      baudRate: 9600
      dataBits: 8
      stopBits: One
      parity: None

  - name: Modbus-TCP-1
    type: ModbusTcp
    enabled: true
    config:
      serverIp: 192.168.1.100
      serverPort: 502
      pollInterval: 1000
      registers:
        - address: 0
          count: 10
          type: HoldingRegister

  - name: MQTT-Sub
    type: Mqtt
    enabled: true
    config:
      broker: mqtt.example.com
      port: 1883
      clientId: gateway-001
      topics:
        - "factory/+/data"
```

#### FR-2: 多协议输出

**需求描述：** 支持向多种协议目标发送数据

**验收标准：**
- 可配置多个输出目标
- 支持负载均衡和故障转移
- 发送失败自动重试（可配置次数）
- 输出目标状态可监控

**配置示例：**
```yaml
outputs:
  - name: MES-API
    type: Http
    enabled: true
    config:
      url: http://mes.example.com/api/data
      method: POST
      headers:
        Authorization: Bearer xxx
        Content-Type: application/json
      retryCount: 3
      timeout: 30000

  - name: AWS-IoT
    type: Mqtt
    enabled: true
    config:
      broker: xxxxx.iot.us-east-1.amazonaws.com
      port: 8883
      useSsl: true
      clientId: gateway-001
      topic: factory/line1/station1

  - name: Local-DB
    type: Sqlite
    enabled: true
    config:
      connectionString: Data Source=data.db
      tableName: measurements
```

#### FR-3: 协议转换

**需求描述：** 支持源协议到目标协议的数据格式转换

**验收标准：**
- 内置常用转换器（SerialToHttp, ModbusToMqtt 等）
- 支持自定义转换器（脚本/插件）
- 转换器可链式组合
- 转换失败有明确错误处理

**内置转换器列表：**

| 转换器 | 源协议 | 目标协议 | 功能 |
|--------|--------|----------|------|
| `SerialToHttp` | Serial | HTTP | 串口数据转 HTTP JSON |
| `ModbusToMqtt` | Modbus | MQTT | Modbus 寄存器值发布到 MQTT |
| `S7ToDatabase` | S7 | Database | 西门子 PLC 数据写入数据库 |
| `JsonTransformer` | JSON | JSON | JSON 字段映射/转换 |
| `CsvParser` | CSV | JSON | CSV 格式解析为 JSON |
| `HexToText` | Hex | Text | 十六进制转文本 |
| `ScriptTransform` | Any | Any | 使用 Lua/JS 脚本自定义转换 |

#### FR-4: 消息路由

**需求描述：** 根据条件将消息路由到不同输出

**验收标准：**
- 支持基于 Topic/主题的路由
- 支持基于消息内容的路由
- 支持基于元数据的路由
- 支持优先级和链式匹配

**配置示例：**
```yaml
routes:
  - name: Temperature-Data
    priority: 10
    condition:
      field: "type"
      operator: "equals"
      value: "temperature"
    outputs:
      - Temperature-DB
      - Temperature-Alert

  - name: Alarm-Data
    priority: 1
    condition:
      field: "level"
      operator: "greaterThan"
      value: 80
    outputs:
      - Alert-System
    continueMatching: false  # 匹配后停止

  - name: Default-Route
    priority: 1000
    condition:
      field: "*"
      operator: "matchAll"
    outputs:
      - Default-DB
```

#### FR-5: 数据过滤与清洗

**需求描述：** 对原始数据进行过滤和清洗

**验收标准：**
- 支持阈值过滤（超出范围丢弃）
- 支持重复数据过滤
- 支持空值/无效值处理
- 支持数据平滑（移动平均等）

**配置示例：**
```yaml
filters:
  - name: Range-Check
    type: RangeFilter
    config:
      field: "value"
      min: 0
      max: 100
      action: discard  # discard/default/custom

  - name: Duplicate-Check
    type: DuplicateFilter
    config:
      windowMs: 1000  # 1 秒内重复数据
      keyField: "deviceId"

  - name: Moving-Average
    type: SmoothingFilter
    config:
      field: "value"
      window: 5  # 5 点移动平均
```

#### FR-6: 配置管理

**需求描述：** 支持配置文件驱动，无需编译

**验收标准：**
- JSON/YAML 配置格式
- 配置文件热加载（修改后自动生效）
- 配置验证（启动前检查配置正确性）
- 配置模板（快速上手）

#### FR-7: 监控与告警

**需求描述：** 实时监控运行状态，异常告警

**验收标准：**
- 输入/输出状态监控
- 消息处理统计（数量、延迟、错误率）
- 告警通知（邮件/短信/钉钉）
- 指标导出（Prometheus 格式）

**监控指标：**
```yaml
metrics:
  - name: gateway_input_messages_total
    type: counter
    labels: [input_name, protocol]

  - name: gateway_output_messages_total
    type: counter
    labels: [output_name, protocol]

  - name: gateway_process_duration_seconds
    type: histogram
    labels: [converter]

  - name: gateway_errors_total
    type: counter
    labels: [component, error_type]
```

### 4.2 数据流设计

```
┌─────────────────────────────────────────────────────────────────┐
│                      数据处理流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Input Plugin → Message → Filter → Transform → Route → Output  │
│       ↓            ↓          ↓          ↓         ↓       ↓    │
│   原始数据    统一格式    过滤清洗    协议转换   路由分发  发送   │
│                                                                 │
│  详细流程:                                                      │
│  ────────────                                                   │
│                                                                 │
│  1. Input Plugin 接收原始数据                                   │
│     ↓                                                           │
│  2. 封装为统一 Message 对象                                      │
│     ↓                                                           │
│  3. Filter Chain 处理                                           │
│     - RangeFilter (范围检查)                                    │
│     - DuplicateFilter (去重)                                     │
│     - TransformFilter (数据清洗)                                │
│     ↓                                                           │
│  4. Converter 处理                                              │
│     - ProtocolConverter (协议转换)                              │
│     - FormatConverter (格式转换)                                │
│     - ScriptConverter (脚本处理)                                │
│     ↓                                                           │
│  5. Router 匹配                                                 │
│     - 根据 Topic 匹配                                            │
│     - 根据内容匹配                                               │
│     - 根据元数据匹配                                             │
│     ↓                                                           │
│  6. Output Plugin 发送                                          │
│     - 负载均衡                                                   │
│     - 失败重试                                                   │
│     - 死信队列                                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 Message 统一数据模型

```csharp
/// <summary>
/// 统一消息模型 - 所有协议转换的核心数据结构
/// </summary>
public class Message
{
    /// 唯一标识
    public string Id { get; set; }

    /// 源协议类型
    public string SourceProtocol { get; set; }

    /// 目标协议类型
    public string TargetProtocol { get; set; }

    /// 主题/路由键
    public string Topic { get; set; }

    /// 原始载荷
    public byte[] Payload { get; set; }

    /// 内容类型 (json/text/hex/binary/modbus/s7/etc)
    public string ContentType { get; set; }

    /// 时间戳
    public DateTime Timestamp { get; set; }

    /// 元数据（协议相关信息）
    public Dictionary<string, string> Metadata { get; set; }

    /// 消息头（用于内部处理）
    public Dictionary<string, object> Headers { get; set; }
}
```
